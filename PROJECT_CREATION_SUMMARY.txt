IMPORTANT NOTES
================================================================================

1. The Admin.cpp file is quite large (~425 lines). For production,
   consider splitting into multiple files (AdminMenu, AdminOperations, etc.)

2. The simple hashing function is for demonstration. For production,
   use MD5, SHA-256, or bcrypt.

3. Console clearing uses system("cls") for Windows. In Linux/Mac,
   change to system("clear").

4. The OTP is displayed in console for demonstration. In production,
   integrate with real SMS/email API.

5. Database format is pipe-delimited text. For scalability,
   use MySQL/PostgreSQL.

6. All timestamps use local system time. For distributed systems,
   use UTC or NTP-synced time.

7. Error handling is basic. For production, add comprehensive
   try-catch blocks and logging.
