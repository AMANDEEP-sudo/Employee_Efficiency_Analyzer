1. FILE DESCRIPTIONS
================================================================================

main.cpp
- Entry point of the application
- Displays main menu (Admin / Citizen / Reports / Exit)
- Handles user input and menu navigation
- Initializes Admin object for core operations

Employee.h / Employee.cpp
- Represents an employee in the system
- Attributes: employee_id, name, department, total_tasks, completed_tasks,
  efficiency_score, feedback_count, task_ids (vector)
- Methods: addTask(), completeTask(), updateRating(), getEfficiencyScore()

Task.h / Task.cpp
- Represents a task assigned to an employee
- Attributes: task_id, employee_id, description, deadline, completion_date,
  is_completed, is_delayed
- Methods: setDeadline(), markComplete(), checkDelay(), isOverdue()

Citizen.h / Citizen.cpp
- Represents a citizen in simulated Aadhaar registry
- Attributes: hashed_aadhaar, name, address, mobile, feedback_ids (vector)
- Methods: validateAadhaar(), hashAadhaar(), addFeedback(), getFeedbackCount()

OTPManager.h / OTPManager.cpp
- Manages OTP generation and verification
- Attributes: otp_store (map), otp_expiry_minutes, max_attempts, attempt_counter
- Methods: generateOTP(), verifyOTP(), checkExpiry(), isMaxAttemptsExceeded()

Feedback.h / Feedback.cpp
- Represents feedback from a citizen for an employee
- Attributes: feedback_id, employee_id, hashed_aadhaar, rating (1-5),
  comment, timestamp, is_submitted
- Methods: setRating(), setComment(), submitFeedback(), displayFeedback()

Department.h / Department.cpp
- Manages department-level operations
- Attributes: dept_name, employee_ids (vector), avg_efficiency
- Methods: addEmployee(), calculateDepartmentAverage(), rankEmployees()

EfficiencyCalculator.h / EfficiencyCalculator.cpp
- Calculates employee efficiency scores based on multiple metrics
- Methods: calculateCompletionRate(), calculateAverageRating(),
  calculateOnTimePercentage(), applyDelayPenalty(), computeFinalScore()

Database.h / Database.cpp
- Handles all file I/O and data persistence
- Methods: saveEmployee(), loadEmployees(), saveTask(), loadTasks(),
  saveCitizen(), loadCitizens(), saveFeedback(), loadFeedback(),
  logAudit(), readAuditLog()

Admin.h / Admin.cpp
- Main control panel managing all system operations
- Attributes: employees (vector), tasks (vector), citizens (vector),
  feedbacks (vector), db (Database object)
- Methods: addEmployee(), assignTask(), viewReport(), generateReport(),
  updateEfficiencyScores(), handleAdminLogin()

Utils.h / Utils.cpp
- Utility functions for common operations
- Functions: inputValidation(), getCurrentDate(), hashString(),
  generateRandomOTP(), displayMenu(), clearScreen()

Constants.h
- Global constants used across the application
- OTP_EXPIRY_MINUTES = 5
- MAX_OTP_ATTEMPTS = 3
- MIN_EFFICIENCY_SCORE = 0.0
- MAX_EFFICIENCY_SCORE = 100.0

3. EFFICIENCY SCORE CALCULATION ALGORITHM
================================================================================

Formula:
efficiency_score = (completion_rate * 50) + (average_rating * 30) 
                   + (on_time_percentage * 20) - delay_penalty

Where:
- completion_rate = completed_tasks / total_tasks
- average_rating = total_rating / feedback_count (normalized to 0-100)
- on_time_percentage = tasks_completed_before_deadline / total_tasks
- delay_penalty = number_of_delayed_tasks * 2

Final Score Range: 0 to 100 (clamped)

Example:
- completion_rate = 0.8 (80%)
- average_rating = 4.2/5 = 0.84 -> 84 (out of 100)  
- on_time_percentage = 0.9 (90%)
- delay_penalty = 2 tasks * 2 = 4

Score = (0.8 * 50) + (84 * 30/100) + (0.9 * 20) - 4
       = 40 + 25.2 + 18 - 4
       = 79.2

5. DATA FILE FORMATS (.dat files)
================================================================================

employees.dat
-----------
Format: id|name|department|total_tasks|completed_tasks|efficiency_score|feedback_count

Example:
1|John Doe|IT|10|8|75.5|5
2|Jane Smith|HR|12|11|82.3|6

tasks.dat
-----------
Format: task_id|employee_id|description|deadline|completion_date|status|is_delayed

Example:
1|1|Complete Report|2024-02-20|2024-02-18|Completed|0
2|1|Code Review|2024-02-25||Pending|0

citizens.dat
-----------
Format: hashed_aadhaar|name|address|mobile

Example:
a1b2c3d4e5f6|Raj Kumar|123 Main St|9876543210
x7y8z9m0n1o|Priya Singh|456 Oak Ave|9123456789

feedback.dat
-----------
Format: feedback_id|employee_id|hashed_aadhaar|rating|comment|timestamp|submitted

Example:
1|1|a1b2c3d4e5f6|5|Excellent service|2024-02-16 10:30|1
2|1|x7y8z9m0n1o|4|Good work|2024-02-16 11:15|1

audit_log.dat
-----------
Format: timestamp|action|user

Example:
2024-02-16 09:00:00|EMPLOYEE_ADDED|admin
2024-02-16 09:05:00|TASK_ASSIGNED|admin
2024-02-16 10:30:00|FEEDBACK_SUBMITTED|citizen_a1b2c3

8. IMPORTANT NOTES
================================================================================

1. OTP Expiry: Set to 5 minutes. Check timestamp before accepting OTP.

2. Aadhaar Hashing: Use string hashing (simple hash or MD5) for 
   simulated Aadhaar storage. Do NOT store plaintext.

3. Duplicate Feedback: Check if feedback already exists for the same 
   citizen-employee pair before allowing submission.

4. Efficiency Calculation: Run after each feedback or task completion 
   to keep scores updated.

5. File I/O: Use binary mode for better performance or text mode
   with pipe-delimited format for human-readable files.

6. Error Handling: Validate all inputs (dates, ratings, IDs, OTP).

7. Menu System: Use switch-case statements for clean menu handling.

8. Data Persistence: Load data at program start, save after each operation.

9. Console Clearing: Use system("clear") on Linux/Mac or 
   system("cls") on Windows.

10. Timestamps: Use ctime library or <chrono> for current date/time.
